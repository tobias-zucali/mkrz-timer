# Internationalization

For product/setup context, start with the [README](../README.md).

## Ownership Rule

Message namespaces follow UI ownership, not file depth.

Supported locales are currently `en` and `de`.
Localized app routes use a required leading locale segment such as `/en`, `/de`, `/en/view/<token>`, and `/de/control/<token>`.

- Use a top-level namespace for a genuinely reusable component or feature boundary.
  Example: `CloseButton`, `StatusBadge`, `Timer`, `TimerPage`.
- Keep nested components and nested utilities inside the parent namespace when they are still owned by that parent domain.
  Example: `Sidebar.settings`, `Sidebar.share`, `Sidebar.status`, `Sidebar.timer`, `Sidebar.participantSummary`.

## File Placement

- Top-level reusable component:
  `src/components/<Component>/messages/<locale>.json`
- Feature boundary:
  `src/features/<Feature>/messages/<locale>.json`
- Nested, feature-owned subcomponents or utilities:
  give them their own nested directory with local `messages/<locale>.json`, but merge that copy back into the parent namespace.

Examples:

- `src/components/Sidebar/messages/en.json` owns the root sidebar copy used by `Sidebar/index.tsx`.
- `src/components/Sidebar/StatusPanel/messages/en.json` sits next to the nested status panel, but those strings are still exposed as `Sidebar.status.*`, not as a new top-level namespace.
- `src/components/Sidebar/participantSummary/messages/en.json` sits next to the nested helper, but those strings are still exposed as `Sidebar.participantSummary.*`.
- `src/features/TimerPage/messages/en.json` owns route-specific copy for `useLiveSessionDialogs.ts`, `useSessionDiagnostics.ts`, and other TimerPage-only utilities.

## Composition

`src/i18n/messages.ts` is the single assembly point for locale bundles. It should import one message file per reusable top-level boundary and expose the merged locale object to `next-intl`.

That means:

- nested sidebar copy is imported from the nested sidebar subdirectories and merged into the `Sidebar` namespace by `src/i18n/messages.ts`
- route-specific timer-page copy is merged by `src/features/TimerPage/messages/en.json`
- `src/i18n/messages.ts` should not recreate ownership decisions that belong closer to the component or feature

Each owned namespace should ship one file per supported locale:

- `messages/en.json`
- `messages/de.json`

English is the default redirect locale for legacy unprefixed URLs such as `/`, `/view/<token>`, and `/control/<token>`.

## Practical Test

Before creating a new top-level namespace, ask:

1. Would this nested file make sense outside the parent component or feature?
2. Would another route or domain reasonably import it without bringing the parent along?

If the answer is no, keep the messages under the parent namespace.
